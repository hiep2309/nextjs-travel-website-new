# VN Insight — Multilingual Architecture (next-intl)

Production i18n for **vi** (default), **en**, **ko**. No Google Translate widget — all copy is native.

---

## 1. Folder structure

```
i18n/
  routing.ts          # locales, defaultLocale, Link/redirect helpers
  request.ts          # server message loader (next-intl plugin entry)

messages/
  vi.json             # UI strings (source of truth)
  en.json
  ko.json

middleware.ts         # locale detection + URL prefix

app/
  layout.tsx          # root passthrough
  [locale]/
    layout.tsx        # html lang, NextIntlClientProvider, AppShell
    page.tsx
    explore/
    destinations/[slug]/   # multilingual SEO example
    ...

lib/i18n/
  types.ts            # LocalizedString, Firestore document shapes
  content.ts          # pickLocalized(), pickSlug(), pickSeo()
  metadata.ts         # buildLocalizedMetadata() + hreflang

components/
  LocaleSwitcher.tsx  # glassmorphism language dropdown
```

## 2. URL routing

| Locale | Example |
|--------|---------|
| vi | `/vi/destinations/ha-noi` |
| en | `/en/destinations/ha-noi` |
| ko | `/ko/destinations/ha-noi` |

- **Fallback:** Vietnamese (`vi`)
- **Detection:** `Accept-Language` via middleware
- **Prefix:** always (`localePrefix: 'always'`)

Use navigation helpers — never hardcode `/vi/...`:

```tsx
import { Link, useRouter } from "@/i18n/navigation";

<Link href="/explore">{t("destinations")}</Link>
router.push("/profile");
```

## 3. UI translations (JSON)

Namespaces in `messages/{locale}.json`:

- `Meta` — site title, description
- `Nav`, `Footer`, `Auth`, `Common`, `Errors`
- `Destinations`, `Posts`, …

**Server component:**

```tsx
import { getTranslations } from "next-intl/server";

const t = await getTranslations("Destinations");
return <h1>{t("title")}</h1>;
```

**Client component:**

```tsx
"use client";
import { useTranslations } from "next-intl";

const t = useTranslations("Nav");
```

## 4. Firebase multilingual schema

Store translations as **maps keyed by locale** (not separate documents per language).

### `posts/{id}`

```json
{
  "sourceLocale": "vi",
  "title": { "vi": "Hà Nội 3 ngày", "en": "3 days in Hanoi", "ko": "하노이 3일" },
  "description": { "vi": "...", "en": "...", "ko": "..." },
  "contentHtml": { "vi": "<p>...</p>", "en": "<p>...</p>" },
  "slugs": { "vi": "ha-noi-3-ngay", "en": "hanoi-3-days", "ko": "hanoi-3-il" },
  "seo": {
    "vi": { "title": "...", "description": "...", "keywords": ["hà nội"] },
    "en": { "title": "...", "description": "..." }
  },
  "translationStatus": {
    "vi": "published",
    "en": "reviewed",
    "ko": "machine"
  }
}
```

### Resolve in code

```tsx
import { pickLocalized, pickLocalizedHtml } from "@/lib/i18n/content";

const title = pickLocalized(post.title, locale);
const body = pickLocalizedHtml(post.contentHtml, locale);
```

Legacy string fields still work — `pickLocalized` falls back automatically.

## 5. SEO metadata

```tsx
import { buildLocalizedMetadata } from "@/lib/i18n/metadata";

export async function generateMetadata({ params }) {
  return buildLocalizedMetadata({
    locale: params.locale,
    path: "/destinations",
    slugs: post.slugs,
    seo: post.seo,
    fallback: { title: "...", description: "..." },
  });
}
```

Generates: `canonical`, `alternates.languages` (hreflang), Open Graph locale.

## 7. AI-assisted translation workflow

1. Author writes in **source locale** (usually `vi`).
2. Export JSON fields → send to LLM with glossary (place names, brand “VN Insight”).
3. Save as `translationStatus: "machine"`.
4. Human editor reviews in admin → `"reviewed"` → `"published"`.
5. Never auto-translate on the live site at request time (latency + SEO).

**Prompt template:**

```
Translate the following travel content from Vietnamese to {targetLocale}.
Keep proper nouns (Hạ Long, Sa Pa). Output JSON only.
Fields: title, description, contentHtml (preserve safe HTML tags).
```

## 8. Admin dashboard workflow

Recommended Firestore admin UI tabs per post:

| Tab | Action |
|-----|--------|
| VI | Edit source |
| EN / KO | Side-by-side editor, status badge |
| SEO | Per-locale title, description, slug |
| Publish | Set `translationStatus[locale] = published` |

Batch job: Cloud Function on `posts` create → queue translation for en/ko.

## 10. Migrating remaining components

1. Replace `next/link` → `@/i18n/navigation` `Link`
2. Replace `useRouter` from `next/navigation` → `@/i18n/navigation`
3. Replace hardcoded strings → `useTranslations` / `getTranslations`
4. Update `generateMetadata` to include `params.locale`
5. Add `setRequestLocale(locale)` in static pages
